Software Development Techniques

Waterfall Development Methodology

The Waterfall model is a sequential approach where each phase must be completed before the next begins. It follows a linear process:

  • Requirement Analysis: Gather and document all stakeholder requirements.
  • Design: Create a system architecture and design plan.
  • Implementation: Developers write code based on the design.
  • Testing: The system is tested for bugs and performance issues.
  • Deployment: The software is released to users.
  • Maintenance: Any issues found are resolved in updates.

Pros: Well-structured, clear deadlines, good for projects with stable requirements.

Cons: Inflexible, difficult to accommodate changes, late testing phase.

Pressman & Maxim, 2020

Agile Development Methodology

The Agile methodology is an iterative approach that emphasizes flexibility, collaboration, and customer feedback. It consists of:

  • Scrum & Sprints: Work is divided into short cycles (sprints) to deliver small, incremental development.
  • Continuous Feedback: Stakeholders review progress frequently and provide feedback.
  • Cross-functional Teams: Developers, testers, and business analysts work together closely.

Pros: Adaptable to changes, faster delivery, continuous testing and feedback.

Cons: Requires strong teamwork, less predictability in timeline and costs.

Cohn, 2004

Secure Data Practices

Ensuring data security is a critical part of modern software development. Some best practices include:

  • Encryption: Encrypt sensitive data during storage and transit.
  • Access Control: Implement role-based access control (RBAC) to restrict sensitive data.
  • Regular Security Audits: Conduct vulnerability assessments and utilise penetration testing.
  • Data Masking: Protect user data by masking personally identifiable information (PII).
  • Use Secure Protocols: Enforce HTTPS and secure API authentication methods.

Following these practices helps prevent data breaches and ensures compliance with regulations like the GDPR.

Cisa, 2024

The Zen of Python

The Zen of Python is a collection of guiding principles for writing clean, readable, and maintainable Python code. Some key principles include:

  • Beautiful is better than ugly.
  • Simple is better than complex.
  • Readability counts.
  • Errors should never pass silently.
  • There should be one—and preferably only one—obvious way to do it.
  • Now is better than never.

Peters,T. 2024
